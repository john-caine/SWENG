Software

Spoon's software team comprised of Steve Thorpe (Software Manager), Jonathan Caine (Development Manager) and Roger Tan (Testing and Integration Manager).

Using Agile methodologies, the software team led the product development process from design to implementation and testing.

All documentation relating to the software side of the project is included below, along with any other documents generated by the software team.

GDS/IDS

After consulting with our Line Manager, Stuart Porter, we were informed that we should only create documentation that the company felt would benefit or be of use to it, not simply generate documentation for the sake of documentation. Consequently following the Agile methodologies that we as a company had adopted, we decided not to produce IDS/GDS documents. This is because they are not a very Agile way of working due to the fact they would inevitably have required constantly changing, due to the continuing refactoring of code. Both the Software Manager and Development Manager, strongly felt that the combination of Test Driven Development and Agile methodologies, meant that IDS/GDS documentation was therefore redundant. Instead the combination of our Functional Specification, and list of user stories were used as a general specification, with the underlying implementation being left open to the programmers assigned to a task, as recommended in the Agile methodologies.

DTP Records

This document contains the deterministic pass test (DTP) record, which was a series of tests (mostly visual) that were to be carried out at the point of integrating a module.

The purpose of these tests was to ensure that by integrating a module or conducting bug fixes, the original functionality of the product was unaltered (not broken!).

Using JIRA, the test results were uploaded to the relevant ticket for review by the testing and integration manager and/or the software manager.

The project DTP records are available here.
